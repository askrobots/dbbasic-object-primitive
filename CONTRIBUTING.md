# Contributing

Thanks for your interest in Object Primitive! This is a research prototype —
we're demonstrating concepts more than running a polished product. Bug
reports, small fixes, and discussion are all welcome.

## Reporting bugs and issues

File them at https://github.com/askrobots/dbbasic-object-primitive/issues.

A good report includes:
- What you ran (command + config, minus any secrets)
- What you expected
- What actually happened (include the full error and a traceback if any)
- OS, Python version, and the commit SHA you're on

## Security issues

**Do not open a public issue for a vulnerability.** See `SECURITY.md` for
the private reporting channel.

## Pull requests

1. Fork the repo and make a branch from `main`.
2. Keep the change focused — one fix or feature per PR.
3. Follow the existing code style; no new dependencies without discussion.
4. If you change behavior, update or add a test (see `tests/`).
5. Run the existing test suite before submitting.

If you're planning a larger change, open an issue first to talk through
the approach — saves everyone time.

## Scope

The runtime itself is deliberately small. Novel object examples and
tooling improvements are easier to land than core-runtime reworks. See
the "Current Status" section of `README.md` for what the project is and
isn't trying to be yet.
